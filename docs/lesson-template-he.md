# תבנית יחידת לימוד

## 1. פרטי יחידה

```text

ידע מוקדם:
תגיות:
גרסת תוכן:
תאריך עדכון:
כותב:
בודק מקצועי:
```

## 2. מטרת היחידה

נסח במשפט אחד מה הלומד ידע לעשות בסוף היחידה.

דוגמה:

```text
בסוף היחידה הלומד ידע לבחור קדח מקדים להברזה מטרית בסיסית ולהסביר למה קדח קטן מדי או גדול מדי יגרום לכשל.
```

## 3. מצב פתיחה מהעולם האמיתי

פתח בתרחיש קצר שמסביר למה הנושא חשוב.

```text
בייצור ראשון של חלק אלומיניום, הברגות M5 נקרעו בזמן הרכבה. בבדיקה התברר שהקדח המקדים לא התאים, עומק ההברזה היה קצר מדי, והבורג תפס מעט מדי תבריג.
```

שאלת פתיחה:

```text
מה היית בודק לפני שאתה מחליט אם הבעיה היא בחומר, בבורג, בקדח או בהרכבה?
```

## 4. הסבר קצר

הסבר את העיקרון בשפה פשוטה. רצוי לא יותר מ-300 עד 500 מילים לפני התרגול הראשון.

מבנה מומלץ:

- מה זה.
- למה זה חשוב.
- איפה פוגשים את זה בעבודה.
- מה הטעות הנפוצה.
- איך חושבים על זה נכון.

## 5. מושגים מרכזיים

```text
מושג:
הסבר קצר:
מונח באנגלית:
איפה רואים את זה:
```

דוגמה:

```text
מושג: קדח מקדים
הסבר קצר: קדח שקודחים לפני הברזה כדי לאפשר לכלי ההברזה ליצור תבריג נכון.
מונח באנגלית: tap drill hole
איפה רואים את זה: בשרטוטי ייצור, טבלאות הברזה והוראות ייצור.
```

## 7. דוגמה מודרכת

תן דוגמה אחת מלאה, מהבעיה עד ההחלטה.

```text
תרחיש:
נתונים:
שיקולים:
החלטה:
מה היה קורה בבחירה שגויה:
```

## 8. טעות נפוצה

```text
הטעות:
למה היא קורה:
איך היא נראית בשטח:
מה הנזק:
איך מונעים:
```

דוגמה:

```text
הטעות: שימוש בשני פינים ובכמה ברגים למיקום שני חלקים בלי לאפשר חופש.
למה היא קורה: ב-CAD הכל מיושר בצורה מושלמת.
איך היא נראית בשטח: החלקים לא נכנסים, או נכנסים רק בכוח.
מה הנזק: הרכבה איטית, שחיקה, עומסים פנימיים וקליבה.
איך מונעים: מגדירים datum אחד, משתמשים בפין מיקום אחד ופין יהלום/slot או חופש מבוקר.
```

## 9. תרגיל אינטראקטיבי

```text
שם התרגיל:
סוג התרגיל:
תרחיש:
נתונים:
שאלה:
תשובה נכונה:
מסיחים:
משוב לתשובה נכונה:
משוב לכל תשובה שגויה:
מיומנות נמדדת:
רמת קושי:
```

סוגי תרגילים אפשריים:

- בחירה מרובה.
- סימון אזור בעייתי בתמונה.
- גרירה וסידור פעולות.
- בחירת רכיב.
- הזנת ערך מספרי.
- תרחיש פתוח לבדיקה אנושית.
- סימולציה פרמטרית.

## 10. משוב לימודי

המשוב צריך להסביר את החשיבה, לא רק לציין נכון או לא נכון.

מבנה משוב:

```text
מה נכון בהחלטה:
מה הסיכון:
מה היה צריך לבדוק:
איך מהנדס מנוסה היה חושב על זה:
```

## 11. המחשה חזותית

```text
סוג נכס:
תיאור:
מטרת הנכס:
מקור או סטטוס הפקה:
דרישות מיוחדות:
```

סוגי נכסים:

- תמונה.
- שרטוט.
- חתך.
- מודל תלת ממדי.
- אנימציה.
- וידאו קצר.
- סימולציה.

## 12. צ'קליסט מעשי

בסוף כל יחידה צריך להיות checklist קצר שהלומד יכול לקחת לעבודה.

דוגמה:

```text
לפני שליחת חלק עם הברזות לייצור:
- האם סוג ההברגה מוגדר?
- האם עומק ההברזה מוגדר?
- האם ברור אם הקדח עובר או עיוור?
- האם יש מספיק אורך תפיסה לבורג?
- האם החומר מתאים להברזה ישירה או שצריך insert?
- האם יש גישה לכלי הברזה?
```

## 15. הגדרת סיום

יחידה מוכנה לפרסום כאשר:

- מטרת הלמידה ברורה ומדידה.
- יש מצב פתיחה אמיתי.
- יש הסבר קצר.
- יש לפחות דוגמה אחת.
- יש לפחות טעות נפוצה אחת.
- יש לפחות תרגיל אחד עם משוב.
- יש המחשה חזותית או דרישה ברורה לנכס חזותי.
- יש checklist.
- מקורות סומנו.
- מומחה מקצועי עבר על התוכן.
